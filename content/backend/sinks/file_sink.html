FileSink

class gpype.FileSink(file_name, input_ports=None, **kwargs)[source]

Bases: INode

Saves incoming data streams to CSV files with automatic timestamping.

The FileSink receives data from input ports and writes it to CSV files in a separate worker thread to avoid blocking the main processing pipeline. Files are automatically timestamped to prevent overwrites and enable easy data organization.

Parameters:

  • file_name (str) – Base filename for the output file. Must have .csv extension. A timestamp will be automatically appended to create the final filename in format: {name}_{YYYYMMDD_HHMMSS}.csv

  • input_ports (list[IPort.Configuration], optional) – Configuration for input ports. Defaults to a single input port.

Examples

>>> # Save single channel data
>>> data_sink = FileSink(file_name="experiment_data.csv")

>>> # Save multi-channel data with custom port configuration
>>> multi_sink = FileSink(
...     file_name="multichannel_data.csv",
...     input_ports=[
...         IPort.Configuration(name="eeg_data"),
...         IPort.Configuration(name="emg_data")
...     ]
... )

Notes

  • Only CSV format is currently supported

  • Files are created with automatic timestamp: basename_YYYYMMDD_HHMMSS.csv

  • Writing happens in a separate thread to maintain real-time performance

  • The sink must be properly started and stopped to ensure data integrity

  • Data is queued and written asynchronously to prevent pipeline blocking
