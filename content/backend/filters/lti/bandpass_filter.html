BandPassFilter

class gpype.BandPassFilter(low_cutoff_frequency, high_cutoff_frequency, sampling_rate, order=4, **kwargs)[source]

Bases: LTIFilter

Butterworth band-pass filter for signal frequency selection.

This filter allows frequencies within a specified band to pass through while attenuating frequencies outside the band. It uses a Butterworth design which provides maximally flat frequency response in the passband and smooth roll-off in the stopband.

The filter is implemented as a cascade of high-pass and low-pass stages, automatically designed using scipy.signal.butter with the ‘bandpass’ type.

Parameters:

  • low_cutoff_frequency (float) – Lower cutoff frequency of the passband in Hz. Frequencies below this will be attenuated.

  • high_cutoff_frequency (float) – Upper cutoff frequency of the passband in Hz. Frequencies above this will be attenuated.

  • sampling_rate (float) – Sampling rate of the input signal in Hz.

  • order (int, optional) – Filter order. Higher orders provide steeper roll-off but may introduce more phase distortion. Defaults to 4.

Examples

>>> # Extract alpha band (8-13 Hz) from EEG at 250 Hz sampling rate
>>> alpha_filter = BandPassFilter(
...     low_cutoff_frequency=8.0,
...     high_cutoff_frequency=13.0,
...     sampling_rate=250.0
... )

>>> # Extract gamma band (30-100 Hz) with higher order
>>> gamma_filter = BandPassFilter(
...     low_cutoff_frequency=30.0,
...     high_cutoff_frequency=100.0,
...     sampling_rate=1000.0,
...     order=6
... )

Notes

  • The passband is defined as [low_cutoff_frequency, high_cutoff_frequency]

  • Cutoff frequencies should satisfy: 0 < low < high < sampling_rate/2

  • Higher filter orders provide steeper roll-off but longer transients
