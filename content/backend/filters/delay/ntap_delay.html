N-Tap Delay

class gpype.NTapDelay(taps, **kwargs)[source]

Bases: IONode

Introduces a configurable N-sample delay to the input signal.

The NTapDelay filter delays the input signal by N samples, where N is the number of ‘taps’. This is commonly used in signal processing for creating delayed versions of signals, implementing FIR filters, or synchronization purposes. The filter maintains an internal buffer to store previous samples.

Parameters:

taps (int) – Number of samples to delay the signal. Must be non-negative. - taps = 0: No delay (immediate passthrough) - taps = 1: Delay by 1 sample - taps = N: Delay by N samples

Examples

>>> # Delay signal by 5 samples
>>> delay = NTapDelay(taps=5)

>>> # No delay (passthrough)
>>> passthrough = NTapDelay(taps=0)

Notes

  • During the initial N samples, the output will be zeros until the buffer is filled

  • The filter requires N+1 samples in its internal buffer before producing delayed output

  • Memory usage scales with the number of taps and channels
