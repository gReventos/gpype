Router

class gpype.Router(input_selector=None, output_selector=None, **kwargs)[source]

Bases: IONode

Routes and redistributes data between input and output channels.

The Router allows flexible routing of input channels to output channels, enabling channel selection, reordering, duplication, and redistribution. It supports multiple input ports and multiple output ports with configurable channel mappings.

Parameters:

  • input_selector (list[list[int]], optional) – Channel selection for each input port. Each inner list specifies which channels to select from that input port. Use Router.ALL (-1) to select all channels. Defaults to [Router.ALL].

  • output_selector (list[list[int]], optional) – Channel routing for each output port. Each inner list specifies which selected channels to route to that output port. Defaults to [Router.ALL].

Examples

>>> # Select channels 0, 2, 4 from input and pass to output
>>> channel_selector = Router(
...     input_selector=[[0, 2, 4]],
...     output_selector=[[Router.ALL]]
... )

>>> # Split 8 channels into two groups of 4
>>> channel_splitter = Router(
...     input_selector=[[Router.ALL]],
...     output_selector=[[0, 1, 2, 3], [4, 5, 6, 7]]
... )

>>> # Duplicate channel 0 to multiple outputs
>>> duplicator = Router(
...     input_selector=[[0]],
...     output_selector=[[0], [0], [0]]
... )

Notes

  • Router.ALL (-1) selects all available channels

  • Channel indices are 0-based

  • Multiple input ports are named “in1”, “in2”, etc.

  • Multiple output ports are named “out1”, “out2”, etc.

  • Channel routing is performed after channel selection
